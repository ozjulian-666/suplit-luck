2. Introduccion
Este manual tecnico documenta la estructura y operacion interna de Suplit Luck para facilitar instalacion, soporte, mantenimiento y evolucion del software.

3. Objetivos del Sistema
El sistema busca:

Centralizar la gestion de usuarios y roles de la plataforma de rifas.
Controlar creacion de rifas, participaciones, compras de boletos y sorteos.
Trazar la operacion por usuario y por rifa activa.
4. Requisitos del Sistema
Infraestructura minima sugerida:

PHP 8.0 o superior.
MySQL 5.7+ o MariaDB 10.4+.
Apache 2.4+.
XAMPP en Windows.
Permisos requeridos:

Escritura en uploads/.
Escritura en imagenes_rifas/.
Escritura en otros directorios de archivos si aplican.
5. Arquitectura del Software
Arquitectura general en tres capas:

Presentacion: vistas HTML/PHP en la raiz del proyecto.
Logica: scripts PHP en programas/ o directamente en archivos PHP.
Datos: esquema MySQL definido en BD.sql.
Componentes principales:

Autenticacion y sesiones.
Gestion administrativa de rifas.
Operacion de usuarios registrados e invitados.
Gestion de archivos en uploads/ e imagenes_rifas/.
6. Instalacion y Configuracion
Instalacion local:

Copiar proyecto a c:/xampp/htdocs/suplit-luck.
Iniciar Apache y MySQL en XAMPP.
Importar BD.sql.
Verificar conexion en programas/comun/conexion.php (o archivo equivalente).
Probar acceso en http://localhost/suplit-luck/1_invitado.php.
Configuracion de BD (conexion.php):

Host: localhost.
User: root.
Password: vacio.
DB: suplit_luck_db (o nombre definido en BD.sql).

9. APIs y Servicios Externos
El sistema no expone API REST publica formal. La operacion se realiza por endpoints PHP internos.

Servicios externos detectados:

No se identifican integraciones obligatorias con terceros en el flujo base.
Carga y gestion de archivos local en servidor web.
10. Seguridad
Riesgos tecnicos detectados:

SQL por concatenacion en varios scripts.
Contrasenas en texto plano.
Control de sesion no uniforme en todos los endpoints.
DDL en tiempo de ejecucion en algunos flujos.
Recomendaciones:

Migrar contrasenas a password_hash/password_verify.
Aplicar prepared statements.
Unificar middleware de autenticacion y autorizacion.
Separar migraciones de base de datos del flujo de negocio.
11. Pruebas y Depuracion
Pruebas funcionales recomendadas:

Alta/edicion de usuario.
Creacion y consulta de rifas.
Compra de boletos y confirmacion.

Depuracion:

Revisar errores de PHP/mysqli.

12. Mantenimiento y Actualizaciones
Lineamientos:

13. Resolucion de Problemas
Incidencias comunes:

Error de conexion a base de datos.
Login sin redireccion.
Error en carga de imagenes de rifas.
Modulos vacios por asociacion incorrecta de usuario.
